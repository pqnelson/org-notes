* Overview

It's still unclear to me the exact origins, but this is how I understand
Lisp machines: basically a RISC processor with some microcoded "Lisp
virtual machine" target. This is morally right, but quite wrong
historically. Correcting the errors, I think, will illuminate and
enlighten.

# https://handwiki.org/wiki/Lisp_machine

** CPU

The [[https://en.wikipedia.org/wiki/Symbolics][Symbolics]] 3600 apparenly used a [[https://en.wikipedia.org/wiki/Motorola_68000_series][Motorola 68k]] CPU, but later with the
introduction of the Ivory and Genera system they moved to the
64-bit [[https://en.wikipedia.org/wiki/DEC_Alpha][Alpha]] architecture.

Xerox Daybreak used the MESA CPU. Apparently this is an esoteric CPU.
- Richard K. Johnsson, John D. Wick, [[http://www.bitsavers.org/pdf/xerox/mesa/princ_ops/An_Overview_Of_The_Mesa_Processor_Architecture_1982.pdf][An Overview of the Mesa Processor Architecture]], 1982

Interlisp used a [[https://en.wikipedia.org/wiki/MOS_Technology_6502][MOS 6502]] CPU.
