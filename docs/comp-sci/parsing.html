1. Pratt Parsing

Pratt parsing shines when you need to parse expressions, and they can have operators with wildly different precedence inserted at any time.

2. Earley Parser

When you need to parse "some" context-free grammar, but don't have any details beyond "it's context-free".

You can also look at LR parsing as an Earley parser restricted to an LR grammar: any \(LR(k)\) grammar can be parsed in \(\mathcal{O}(n)\) time (where \(n\) is the length of the input string).

The basic idea is to take your input string and work one token (terminal symbol) at a time. At position \(j\) we build up a set of items \(I_{j}\) representing the state of production used in a possible derivation.

An Earley item looks like \((A\to\beta\bullet\gamma,k)\) which consists of

  1. the production rule \(A\to\beta\gamma\) with a dot located on the right-hand side to show how much has been used thus far; and
  2. the position \(k\) in the input where the parsing of the nonterminal \(A\) started (in terms of the number of tokens read up to that point).

That's then used to build up all possible derivations. It starts by initializing the set \(I_{0}\) to the rule \((S'\to\bullet S,0)\) where \(S\) is the start symbol for the grammar and \(S'\) is a new artificial symbol introduced to simpify the algorithm. If the algorithm successfully parses the input \(a_{0}a_{1}\cdots a_{n-1}\), then the item \((S'\to S\bullet,0)\) appears in the set \(I_{n}\).

There are three possible rules guiding the algorithm:

  • Scan: if the set \(I_{j}\) contains \((A\to\beta\bullet c\delta,k)\) where \(c=a_{j}\), then the item \((A\to\beta c\bullet\delta,k)\) is added to \(I_{j+1}\). These can be done as the final actions at a given input position since they do not affect \(I_{j}\)
  • Predict: if \(I_{j}\) contains an item \((A\to\beta\bullet C\delta,k)\), then for each production rule \(C\to\gamma\) in the grammar, an item \((C\to\bullet\gamma,j)\) is added to \(I_{j}\); this can propagate to require more predict steps, when \(\gamma\) starts with a nonterminal.
  • Complete: If the set \(I_{j}\) contains a completed item \((C\to\gamma\bullet,k)\), then for each item of the form \((A\to\beta\bullet C\delta,\ell)\) in \(I_{k}\), we add an item \((A\to\beta C\bullet\delta,\ell)\) to \(I_{j}\).

The only problem is that, for ambiguous grammars, this gives us a recognizer (i.e., a function which returns true if the string belongs to the language and false otherwise), not a parser (i.e., a function which constructs some kind of syntax tree).

We could choose to work with "whatever parses first", but there are other hacks to make it into a parser.
